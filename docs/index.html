Getting Started

Setup & Configuration

Two files control all configuration: .env for secrets and config.py for app settings. Never commit .env to source control.

.env — secrets & credentials
# ── SAP BO Server ───────────────────────
BO_HOST      = your-bo-server.company.com
BO_PORT      = 6405
BO_USERNAME  = Administrator
BO_PASSWORD  = your_password_here
BO_AUTH_TYPE = secEnterprise

# ── AI API Key ───────────────────────────
# Get free key → makersuite.google.com
GEMINI_API_KEY = AIzaSy_paste_your_key_here
BO_AUTH_TYPE — authentication options
# SAP BO Enterprise accounts (default)
BO_AUTH_TYPE=secEnterprise

# Windows Active Directory / SSO
BO_AUTH_TYPE=secWinAD

# LDAP directory
BO_AUTH_TYPE=secLDAP

# SAP system authentication
BO_AUTH_TYPE=secSAPR3

# ── CMS port is usually ─────────────────
# BI 4.x → 6405
# Older   → 6400
# Check: CMC → Servers → CMS → Properties
Source install — Python 3.10+ required
# 1. Clone and enter the folder
git clone https://github.com/<your-repo>/bo-commander
cd bo-commander

# 2. Install dependencies
pip install -r requirements.txt

# 3. Copy the example config and fill it in
copy .env.example .env
#  → Edit .env with your BO server details + Gemini API key

# 4. Launch
python bo_commander.py

FAQ

Frequently Asked Questions

Facing an issue or want to understand how something works? Can't find your answer? Reach out directly →

The app launches but shows 0 everywhere — is it working? ›

Yes — this is correct on an empty BO environment. Every page queries your live CMS, so if there are no reports or instances yet, you'll see zeros. Try Users or Servers first — those will always have data if you're connected.

If you see zeros on Servers too, check that BO_HOST, BO_PORT, BO_USERNAME, and BO_PASSWORD in your .env file are correct, and that port 6405 is reachable from your machine.

Where do I get a Gemini API key and where do I put it? ›

Get a free key from Google AI Studio. The free tier (Gemini 1.5 Flash) is enough for normal admin use. Then add this line to your .env file:

GEMINI_API_KEY=AIzaSy...your_key_here

Without a key, all non-AI features (reports, users, servers, connections, etc.) still work normally — only AI Sentinel, AI Assistant, and Self Healing require it.

License keys are valid for 30 days. When expired, the activation dialog reappears on next launch. To renew:

Which SAP BO versions does this support? ›

BO Commander is tested against SAP BO BI Platform 4.x (4.1 through 4.4). The REST API and CMS query syntax is consistent across all 4.x versions. Some REST endpoints (like /connections/{id}/test) may not exist on older versions — when that happens the tool automatically falls back to a CMS lookup rather than failing.

Is it safe to use on a production BO server? ›

Read operations (browsing reports, users, sessions, running CMS queries, viewing connections) are 100% read-only and safe in production.

Write operations that modify data are always clearly labelled and require confirmation: Delete, Bulk Delete, Run/Refresh Report, Create Connection, Kill Session, Restart Server. Never perform these without reviewing the impact first.

How do I connect to SAP BW, HANA, or S/4HANA? ›

Use the BW Connections or Connection Manager page in the sidebar. BO Commander handles all 4 source types:

  • SAP BW — OLAP (BICS / MDX) — no universe needed, direct InfoProvider & BEx Query access
  • SAP HANA OLAP — Calculation Views with Star Join — no universe needed
  • SAP HANA Relational — via JDBC/ODBC, universe required
  • S/4HANA — BW Query, HANA Calc View, or CDS View
  • Non-SAP — Oracle, SQL Server, MySQL, PostgreSQL, DB2, Teradata

If you get a consistently wrong or harmful answer, report it to saitejaguddanti999@gmail.com with the question and response — this helps improve the guardrails.

Is my BO password stored securely? ›

Credentials are stored in the .env file on your local machine — standard practice for desktop admin tools. The file is never uploaded or transmitted anywhere by BO Commander. Ensure your Windows user account is the only one with read access to the .env file.

For a passwordless setup, leave BO_PASSWORD empty — the login dialog will prompt for it each time without saving to disk.
